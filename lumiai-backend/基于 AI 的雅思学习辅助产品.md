基于 AI 的雅思学习辅助产品
一、 产品核心功能设计
小组讨论的核心围绕如何利用 AI 技术提升雅思听力和阅读的备考效率：
- 错题分析与个性化推荐：
  - 系统会根据学生错题的文本和给出的错误答案，分析其背后的错误原因。
  - 根据错误分类，系统会自动从题库中抽取类似的题目进行针对性练习。
  - 设定过关指标（如连对三道题视为掌握该知识点）。
- 长难句与语法分析：
  - 利用 Python 进行分词，提取主谓宾等语法结构，或直接调用大模型（LLM）进行长难句解析。
- 生词管理系统：
  - 学生在系统中做题时可随时点击标记生词。
  - 系统自动生成包含生词原型、释义、词性及文中例句的单词表。
- 学情分析报告：
  - 定期（如每 7 天）生成分析报告，展示学生在不同题型上的错误分布和学习进度。

二、 商业模式与盈利策略
讨论明确了“AI 工具 + 社媒 IP”的双轨制商业模式：
具体策略
引流与 IP 打造	将小组成员打造为小红书上的“明星老师”IP，通过素人博主发布软广，利用社媒流量为产品引流。
付费模式 (ToC)	采用会员制。免费功能包括模拟考试工具（类似雅思哥）；付费功能包括个性化题库推荐、错题分析和学情报告 。
B 端扩展	未来可向课外辅导机构提供该系统，帮助机构向家长展示学生的学习数据和进步情况。
增值服务	销售配套学习资料、网课、一对一教学，甚至延伸至留学文书咨询服务。

三、 技术方案与 Demo 演示规划
为了应对即将到来的产品演示（Pre），小组制定了务实的开发计划：
- 技术栈选择：
  - 使用 API 成本较低的 Lite 大模型。
  - 利用 Google 开发者或 Google Cloud 的免费额度获取算力支持。
- Demo 演示逻辑：
  - 演示重点：重点展示“阅读”题型的交互，因为听力演示较为复杂且时间有限（总时长 20 分钟）。
  - 交互逻辑：采用“硬逻辑”演示，即预设好特定的点击流程和界面跳转，以保证演示的优雅和稳定性，不强求实时生成。
- UI 设计风格：
  - 追求简洁、优雅、苹果风（Apple Style）。
